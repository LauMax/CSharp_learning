# Git

## 如何去掉远端repo
```git
git remote remove origin
```

## 重新添加想要的远程地址
```
git remote add origin https://xxxxx
```


## 查看当前远程仓库配置
```
git remote -v
```

## 只是想修改远程地址，也可以直接用
```
git remote set-url origin https://xxxx
```


## 撤销最近一次提交，保留所有文件的修改状态（仍在暂存区）
```
git reset --soft HEAD~1
```

## 如果你想清空暂存区（文件仍在工作区）
```
git reset --mixed HEAD~1
```

##  如果想彻底清空所有修改
```
git reset --hard HEAD~1
```


## 重置整个仓库，删除.git 目录， 彻底清空Git历史
```
rm -r .git
```


# Git 中几个区的关系

在 Git 中，整个版本控制的魔法其实围绕着三个核心“区域”展开：工作区（Working Directory）、暂存区（Staging Area）、和版本库（Repository）。我们可以用一个“厨房做饭”的比喻来形象理解它们：

## 🍳 1. 工作区（Working Directory）
——你的厨房操作台
这是你正在“做菜”的地方，也就是你打开文件、修改代码的真实目录。

 - 你在这里切菜、炒菜、加调料（修改文件）

 - 文件还没准备好上桌（提交），只是你在本地动手的状态

🧠 技术上：这是你项目的文件夹，包含当前可见的代码和资源。

## 🧺 2. 暂存区（Staging Area）
——你的上菜托盘
你把做好的菜（文件）放到托盘上，准备端给客人（提交到版本库）。

 - 用 git add 把文件从工作区放到暂存区

 - 暂存区是你“打包好”的提交内容清单

🧠 技术上：这是一个中间缓冲区，Git 会根据这里的内容生成下一次提交。

## 📦 3. 版本库（Repository）
——你的冰箱或菜单记录
你把菜端上桌（提交），并记录在菜单上（版本历史）。

- 用 git commit 把暂存区的内容永久记录下来

- 每次提交就像在菜单上加一道新菜，可以随时翻看、回退、对比

🧠 技术上：这是 .git 文件夹中的数据库，记录所有提交历史、分支、标签等。

🧭 补充：远程仓库（Remote Repository）
——云端菜单展示
比如 GitHub、GitLab，它是你把菜单上传到网上，别人也能点菜（协作开发）。

 - 用 git push 把本地提交同步到远程

 -  用 git pull 把别人的更新拉下来
